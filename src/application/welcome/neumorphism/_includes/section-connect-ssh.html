Connect to Challenges Using SSH

To participate in the CTF challenges, you might need to securely connect to the challenge environments using SSH. This process involves installing the step tool, bootstrapping the Certificate Authority (CA), and issuing an ephemeral SSH certificate. Follow the steps below to get started:

  1. Install step: Begin by installing the step tool: 
    wget https://dl.smallstep.com/cli/docs-cli-install/latest/step-cli_amd64.deb
sudo dpkg -i step-cli_amd64.deb
  2. Bootstrap the CA: Once step is installed, bootstrap the Certificate Authority (CA) by running the following command: 
    step ca bootstrap \
  --ca-url https://{{site.ssh_step_ca_host}} \
  --fingerprint {{site.ssh_step_ca_fingerprint}} \
  --force \
  --install
eval $(ssh-agent)
  3. Issue an SSH Certificate: After bootstrapping the CA, you can issue an ephemeral SSH certificate by executing the command: 
    step ssh login
    Use the issued SSH certificate to connect to the challenge environment. The certificate will be valid for 24 hours. After 24 hours, your certificate will expire. To continue accessing the challenges, simply log in again to renew your certificate.
  4. Accessing Challenge Environments: To access the challenge environments, the virtual machines (VMs) running the challenges are hidden behind an SSH bastion host. Users must perform an SSH proxy jump through the bastion to reach their specific challenge environments. This adds a layer of security by ensuring that all connections pass through a secure intermediary. Below is an example of an SSH configuration to achieve this: 
    Host bastion
    HostName bastion.example.com
    User your_username

Host challenge
    HostName challenge.internal
    ProxyJump bastion
    User your_username
    IdentityFile ~/.ssh/id_ecdsa_cert.pub
